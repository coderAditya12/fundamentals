# The Big Picture

```text
+----------------------+
|    Source Code (.py) |
|  (Human-Readable)    |
+----------------------+
           │
           ▼
+----------------------+
|      Compiler        |
| (Lexical Analysis,   |
| Parsing, AST,        |
| Bytecode Generation) |
+----------------------+
           │
           ▼
+----------------------+
|       Bytecode       |
| (.pyc in __pycache__)|
| (Intermediate Form)  |
+----------------------+
           │
           ▼
+----------------------+
| Python Virtual       |
| Machine (PVM)        |
| Executes Bytecode    |
+----------------------+
           │
           ▼
+----------------------+
| Operating System     |
| + Hardware           |
+----------------------+
```

Python follows a layered architecture that converts human-readable code into instructions that the computer can execute.

Unlike languages such as C or C++, Python does **not** compile directly into machine code. Instead, it first compiles source code into **bytecode**, which is then executed by the **Python Virtual Machine (PVM)**.

---

# Step 1: Source Code (.py)

Everything begins with a Python source file.

Example:

```python
def greet(name):
    return f"Hello, {name}!"

print(greet("Rohan"))
```

The `.py` file is plain text written by the developer.

The CPU cannot understand this code directly.

---

# Step 2: Compilation

When you execute the program,

```bash
python app.py
```

Python first compiles the source code.

Compilation consists of several stages.

---

## 2.1 Lexical Analysis

The source code is broken into tokens.

Example:

```python
x = 10
```

Tokens become

```text
Identifier : x
Operator   : =
Number     : 10
```

---

## 2.2 Parsing

The parser checks whether the syntax is valid.

Example:

```python
if x > 10:
    print(x)
```

Valid.

Example:

```python
if x > 10
    print(x)
```

Invalid.

The parser reports a syntax error because the colon (`:`) is missing.

---

## 2.3 AST (Abstract Syntax Tree)

After successful parsing, Python builds an **Abstract Syntax Tree (AST)**.

Example:

```python
x = 5 + 3
```

Simplified AST

```text
Assignment
│
├── Variable(x)
│
└── Addition
     ├── 5
     └── 3
```

The AST represents the logical structure of the program.

---

## 2.4 Bytecode Generation

The compiler converts the AST into bytecode.

Bytecode is a low-level instruction set designed for the Python Virtual Machine.

Example:

```python
def greet(name):
    return f"Hello {name}"
```

Viewing the generated bytecode:

```python
import dis

def greet(name):
    return f"Hello {name}"

dis.dis(greet)
```

Example output:

```text
LOAD_CONST
LOAD_FAST
FORMAT_VALUE
BUILD_STRING
RETURN_VALUE
```

These instructions are understood by the Python Virtual Machine.

---

# Step 3: Bytecode (.pyc)

After compilation, CPython may save the bytecode inside the `__pycache__` directory.

Example:

```text
project/

main.py

__pycache__/
    main.cpython-312.pyc
```

The purpose of `.pyc` files is to avoid recompiling unchanged source code every time the program runs.

> **Note:** `.pyc` files are specific to the CPython implementation and are tied to a particular Python version.

---

# Step 4: Python Virtual Machine (PVM)

The Python Virtual Machine executes the bytecode.

It reads one bytecode instruction at a time and performs the corresponding operation.

Example:

```text
LOAD_CONST

↓

LOAD_FAST

↓

CALL

↓

RETURN_VALUE
```

The PVM itself is a software program written primarily in C (in CPython).

---

# Step 5: Operating System

Whenever Python needs to perform system-level tasks, it asks the operating system.

Examples:

* Read a file
* Write to disk
* Create a process
* Allocate memory
* Open a network socket

Example:

```python
with open("data.txt") as file:
    print(file.read())
```

The PVM does not access the disk directly. It delegates the operation to the operating system.

# Why Does Python Use Bytecode?

Advantages:

* Faster startup for previously compiled modules.
* Platform-independent intermediate representation.
* Separates compilation from execution.
* Allows different Python implementations to execute Python programs.

---

# Compiler and Interpreter Working Together

Python uses **both** a compiler and an interpreter.

The compiler converts source code into bytecode.

```text
Source Code

↓

Compiler

↓

Bytecode
```

The interpreter (PVM) executes that bytecode.

```text
Bytecode

↓

Python Virtual Machine

↓

Program Output
```

---

# Common Misconceptions

### ❌ Python is purely interpreted.

Not exactly.

Python first **compiles** source code into bytecode.

The bytecode is then interpreted by the Python Virtual Machine.

---

### ❌ Python executes `.py` files directly.

Incorrect.

The `.py` file is compiled into bytecode before execution.

---

### ❌ `.pyc` files contain machine code.

Incorrect.

`.pyc` files contain **Python bytecode**, not CPU instructions.

---

### ❌ Bytecode is universally compatible.

Not always.

Bytecode is generally compatible only with the same Python implementation and compatible Python versions.
